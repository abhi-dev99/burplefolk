[2] 🟠 FIX: Add Stratified Sampling (Not Just LIMIT)
─────────────────────────────────────────────────────
PROBLEM:
  - Currently: SELECT * FROM table LIMIT N (first N rows only)
  - Biases results if data is sorted/partitioned
  - Temporal data shows only oldest records

LOCATION: nexus/ingestion.py (lines 47-50, csv_ingest)

SOLUTION:
  1. For CSV: Use pandas.DataFrame.sample(frac=0.1, random_state=42) with stratification
  2. For SQLite: Use ORDER BY RANDOM() LIMIT N instead of sequential LIMIT
  3. For DB connections: Add sampling strategy parameter (RANDOM, STRATIFIED, RECENT)
  4. Keep profile_row_limit as target sample size, not hard limit

FILES TO CHANGE:
  - nexus/ingestion.py: csv_ingest(), sqlite_ingest(), database_ingest()
  - app.py: Add "Sampling Strategy" dropdown (RANDOM / SEQUENTIAL / RECENT)

HACKATHON WIN:
  Better data representation = more accurate quality scores
